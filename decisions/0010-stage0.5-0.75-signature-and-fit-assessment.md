# 0010 — Stage 0.5/0.75 signature classification and fit assessment

**Stage:** 0.5 / 0.75
**Status:** provisional-accepted
**Date:** 2026-07-12

## Context

ADRs 0002 and 0007 require Stage 0.5 classification and Stage 0.75 fit
assessment to gate confirmatory real-data one-dimensional Stage 2 claims. The
package must classify applicability without allowing the Stage 2 estimator to
justify itself, and must remain general across disease, development, sex
determination, treatment, and other biological contrasts.

The current contracts are deferred and unexported (Issue #5). No signature,
classifier, or fit-assessment algorithm has yet been implemented or benchmarked.

## Options considered

| Option | Key property | Disqualifier or concern |
|---|---|---|
| Disease/domain-labelled classifier | Familiar worked examples | Bakes application ontology into a general package. |
| Force the nearest archetype | Always returns an answer | Converts uncertainty into unsupported Stage 2 eligibility. |
| Signature-distance baseline | Transparent and auditable | May lack flexibility for complex signatures. |
| Flexible `ranger` classifier | Can learn non-linear boundaries | Requires held-out calibration and can be overconfident. |
| Use Stage 2 topology in the signature | Potentially predictive | Circular: the estimator would select its own applicability. |

## Criteria

- Archetypes are geometric and domain-neutral.
- Signatures use only pre-Stage-2 information: Stage 1 geometry,
  layer-response profiles, sampling design, and estimator-independent
  distribution descriptors.
- Classifier selection, fit assessment, and abstention are calibrated on
  held-out simulations, separate from library/training simulations.
- The decision rule prioritises avoiding false eligibility for confirmatory
  one-dimensional Stage 2 claims over raw classification accuracy.
- Classification can return uncertain or out-of-library rather than a forced
  label.
- Biological interpretation is an explicit user-supplied domain mapping.

## Evidence

No Stage 0 archetype library or held-out calibration evidence yet exists. This
is a protocol decision made before implementation. The current one-dimensional
Stage 2 estimator cannot represent branching, multi-axis, or discordant
multi-layer structures.

## Decision

**Chosen:** build a domain-neutral archetype library and compare two Stage 0.5
classifier strategies: a transparent signature-distance baseline and an
optional `ranger` classifier. Use held-out calibration, abstention behaviour,
out-of-library detection, and false-eligibility control to select the default.

Stage 0.75 independently assesses fit/coverage using held-out simulations. The
applicability rule is asymmetric: uncertain support must abstain rather than
admit an unsupported real dataset to a confirmatory one-dimensional Stage 2
claim. Generic archetype labels are interpreted only through an explicit,
user-supplied domain mapping that is provenance-recorded.

## Consequences

- Implement neither classifier nor fit assessment before the Phase 6 Stage 0
  library controls exist.
- The initial archetype library must include ordered one-dimensional
  multistability, single-well/continuous, branching/multi-axis, discordant
  multi-layer, and out-of-library cases.
- Stage 0.5/0.75 outputs become required for confirmatory real-data claims but
  do not prevent clearly labelled exploratory work.
- `ranger` remains optional until evidence selects it over the transparent
  baseline.

## Review trigger

Revisit if held-out controls cannot reliably control false eligibility, if the
pre-Stage-2 signature cannot distinguish ordered one-dimensional from branching
or discordant cases, or if a new classifier strategy improves calibration under
the same frozen protocol.
