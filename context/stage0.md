# Stage 0 — Synthetic control ladder

Stage 0 generates synthetic datasets with known quasi-potentials and uses them to validate that Stages 1 and 2 correctly recover the ground truth. It is load-bearing: Stage 2 has no real-data ground truth, so Stage 0 is the only honest calibration source. The `[tbd]` acceptance thresholds in ADR 0002 cannot be filled until Stage 0 results exist.

## Language

**ground truth**:
A synthetic dataset generated by Stage 0 with a fully specified quasi-potential (known well locations, barrier heights, diffusion coefficient). The evidence oracle for all downstream threshold decisions.
_Avoid_: test data, mock data, simulated data (too generic — ground truth implies the potential is *known*, not merely *simulated*)

**synthetic control**:
A single generated dataset from Stage 0, corresponding to one point in the parameter space of the control ladder.
_Avoid_: fake data, toy dataset

**control ladder**:
The full structured sweep of Stage 0 synthetic controls: varying number of wells, well separation, noise level, sample count, time point density, and rank-deficiency. Each rung of the ladder is a different experimental condition.
_Avoid_: parameter sweep, grid search

**recovery benchmark**:
A Stage 0 experiment that runs the full Stage 1 + Stage 2 pipeline on a synthetic control and measures how accurately Stage 2 recovers the known critical points and barrier heights.
_Avoid_: accuracy test, validation experiment

**thinness sweep**:
The dimension of the control ladder that varies sample count (number of observations) and time point density to find the minimum data requirements for reliable quasi-potential recovery.
_Avoid_: sample-size sensitivity analysis, small-N experiment

**rank-deficiency axis**:
The dimension of the control ladder in which one omic layer's matrix is deliberately constructed with rank less than full column rank. Tests whether Stage 1's HO-GSVD handles this correctly — critical because the diabetes genotype layer will be rank-deficient in production.
_Avoid_: degenerate case, singular matrix test

**acceptance threshold**:
A numeric criterion (e.g. maximum tolerated error in recovered barrier height) derived from Stage 0 recovery benchmarks and recorded in ADR 0002. Stage 2 is not fully accepted until these thresholds are filled in.
_Avoid_: tolerance, cutoff, `[tbd]` (the placeholder — these must become real numbers)

**double-well control**:
A Stage 0 synthetic control whose ground-truth quasi-potential has exactly two wells — the simplest non-trivial case and the baseline validation target.
_Avoid_: two-state control

**three-well control**:
A Stage 0 synthetic control whose ground-truth quasi-potential has three wells — required to validate Stage 2 on CML-like landscapes.
_Avoid_: tri-stable control

### Ground truth classes

**SubspaceGroundTruth**:
A `GroundTruth` subclass recording the known subspace structure (e.g. which singular vectors correspond to the disease axis) for Stage 1 validation.

**TopologyGroundTruth**:
A `GroundTruth` subclass recording the known topology of the quasi-potential (number of wells, well locations, barrier heights) for Stage 2 validation.

**PotentialGroundTruth**:
A `GroundTruth` subclass recording the known analytic quasi-potential function for continuous comparison (not just discrete critical-point comparison).
