# Stage 0 — Synthetic control ladder

Stage 0 generates synthetic datasets with known quasi-potentials and uses them to validate that Stages 1 and 2 correctly recover the ground truth. It is load-bearing: Stage 2 has no real-data ground truth, so Stage 0 is the only honest calibration source. The `[tbd]` acceptance thresholds in ADR 0002 cannot be filled until Stage 0 results exist.

## Language

**ground truth**:
A synthetic dataset generated by Stage 0 with a fully specified quasi-potential (known well locations, barrier heights, diffusion coefficient). The evidence oracle for all downstream threshold decisions.
_Avoid_: test data, mock data, simulated data (too generic — ground truth implies the potential is *known*, not merely *simulated*)

**generic mathematical control**:
A Stage 0 control with planted mathematical truth used to test recovery, false positives, confounding, missingness, and alignment independently of one biological application.

**domain-grounded control**:
A Stage 0 control whose observation structure reflects a planned application, such as AML, *Pogona*, or diabetes layer sizes, noise/count properties, batch patterns, missingness, and sampling design. It tests robustness in realistic conditions but does not replace generic ground-truth recovery evidence.

**biological exemplar**:
A real experimental dataset used only after the relevant synthetic contract and recovery work to demonstrate operability, diagnostics, face validity, and usability. Without external latent-axis ground truth it cannot establish recovery correctness or tune already-frozen acceptance criteria. Capability evidence therefore follows a three-rung ladder: generic synthetic control → domain-grounded synthetic control → biological exemplar. Implementation completion, synthetic scientific validation, and biological operability remain separate states.

**trajectory archetype**:
A domain-neutral geometric class in the Stage 0.5 library, such as ordered one-dimensional multistability, single-well/continuous, branching/multi-axis, discordant multi-layer, or out-of-library. Biological interpretations are applied separately after classification. Stage 0.5 may abstain with an uncertain or out-of-library result rather than force the nearest archetype. Its classifier compares a transparent signature-distance baseline with an optional flexible strategy; held-out calibration and abstention performance choose the default. Signatures use only pre-Stage-2 geometry, layer-response, sampling-design, and estimator-independent distribution descriptors.

**synthetic control**:
A single generated dataset from Stage 0, corresponding to one point in the parameter space of the control ladder.
_Avoid_: fake data, toy dataset

**control ladder**:
The full structured sweep of Stage 0 synthetic controls: varying number of wells, well separation, noise level, sample count, time point density, and rank-deficiency. Each rung of the ladder is a different experimental condition.
_Avoid_: parameter sweep, grid search

**end-to-end recovery benchmark**:
A mandatory Stage 0 acceptance-gate experiment that runs synthetic multi-omic data through Stage 1 and then Stage 2. It measures both target-biological-axis recovery and quasi-potential recovery against planted ground truth before AML or other real-data analysis.
_Avoid_: accuracy test, validation experiment

**potential-recovery benchmark**:
A Stage 0 estimator-level experiment that supplies known coordinates directly to Stage 2 and measures quasi-potential recovery. It validates Stage 2 in isolation; it is not evidence that the full pipeline recovers a known quasi-potential.

**confounder-separation control**:
A mandatory end-to-end synthetic control with a planted target biological axis and a stronger shared confounder, so the target is not the dominant Stage 1 component. It validates axis recovery, component-selection-proposal ranking, and downstream quasi-potential recovery together.

**thinness sweep**:
The dimension of the control ladder that varies sample count (number of observations) and time point density to find the minimum data requirements for reliable quasi-potential recovery.
_Avoid_: sample-size sensitivity analysis, small-N experiment

**rank-deficiency axis**:
The dimension of the control ladder in which one omic layer's matrix is deliberately constructed with rank less than full column rank. Tests whether Stage 1's HO-GSVD handles this correctly — critical because the diabetes genotype layer will be rank-deficient in production.
_Avoid_: degenerate case, singular matrix test

**missingness control**:
A Stage 0 control that masks required omic measurements either at random or in association with a biological variable such as condition or time. It measures the sensitivity of complete-paired-cohort fitting and is required before projection-only use of incomplete observations. In real data, condition/time-associated missingness downgrades claims to exploratory unless a predeclared complete-case sensitivity analysis is stable.

**negative control**:
A Stage 0 control with no target signal, no transition barrier, or deliberately discordant omic layers. It measures false-positive target-axis, topology, and pooling claims; it is a mandatory acceptance gate rather than an optional demonstration.

**delivery gate**:
The stop rule that prevents new Stage 2 capabilities and confirmatory real-data claims until provenance and multi-omic alignment contracts are repaired and the frozen Stage 0 positive/negative control ladder has finalized ADR 0002 acceptance thresholds.

**acceptance threshold**:
A numeric criterion (e.g. maximum tolerated error in recovered barrier height) derived from Stage 0 recovery benchmarks and recorded in ADR 0002. It is evaluated over a predeclared independent replicate set, with an error distribution and pass rate—not a selected seed. Its frozen evidence is a versioned compact benchmark-result table plus generator specification, seed manifest, configuration digest, and environment record; the synthetic matrices are regenerated on demand. Changed evidence produces a new versioned artifact and ADR amendment rather than overwriting prior evidence. Stage 0.75 fit assessment is calibrated on held-out simulations, distinct from signature-library/training simulations. Stage 2 is not fully accepted until these thresholds are filled in.
_Avoid_: tolerance, cutoff, `[tbd]` (the placeholder — these must become real numbers)

**double-well control**:
A Stage 0 synthetic control whose ground-truth quasi-potential has exactly two wells — the simplest non-trivial case and the baseline validation target. Must be validated for K=1 before any K=1 real-data analysis (AML, CML mRNA, Pogona).
_Avoid_: two-state control

**three-well control**:
A Stage 0 synthetic control whose ground-truth quasi-potential has three wells — required to validate Stage 2 on CML-like landscapes.
_Avoid_: tri-stable control

**bifurcation control**:
A Stage 0 synthetic control whose ground-truth topology is a Y-shape: one shared early attractor that splits into two distinct wells as a function of a developmental or condition variable. Characterises the Pogona TSD dataset (undifferentiated early embryos → male/female attractors). This topology is outside the current 1D KDE Stage 2 estimator scope and requires a separate ADR before real-data use.
_Avoid_: branching control, split control

**generic K=1 cross-sectional double-well control**:
A single-layer control with independent observations and a known double-well target coordinate. It validates registered `svd` recovery and the current cross-sectional Stage 2 estimator without AML-specific sampling complexity.

**AML-grounded K=1 longitudinal confounder-separation control**:
A single-layer control with repeated synthetic mice, CTL/CM groups, a planted time effect stronger than the target signal, and planted condition-by-time disease divergence. The nuisance axis dominates PC1 and the target biological axis appears at PC2. It validates the sampling-design-aware atlas, component proposal, biological-unit bootstrap, and alignment. Under ADR 0006 the current cross-sectional Stage 2 estimator must return typed ineligibility rather than pool repeated observations. Load-bearing for AML Stage 1 recapitulation.
_Avoid_: confounded control (too generic)

### Ground truth classes

**SubspaceGroundTruth**:
A `GroundTruth` subclass recording the known subspace structure (e.g. which singular vectors correspond to the target biological axis) for Stage 1 validation.

**TopologyGroundTruth**:
A `GroundTruth` subclass recording the known topology of the quasi-potential (number of wells, well locations, barrier heights) for Stage 2 validation.

**PotentialGroundTruth**:
A `GroundTruth` subclass recording the known analytic quasi-potential function for continuous comparison (not just discrete critical-point comparison).
