## The Rockne-Frankhouser algorithm

We read the actual MATLAB code from `cohmathonc/CML_mRNA_state-transition`.
The Stage 2 algorithm is:
1. KDE over PCA/SVD coordinates → p(x)
2. U(x) = −log p(x) at critical points (hard-coded by hand in the MATLAB)
3. Constrained 6th-degree polynomial: simultaneously fit U values AND force
   gradient to zero at critical points — one linear system, solved with `\`
4. Barrier height = polynomial value at saddle minus value at well
5. Optional Fokker-Planck forward propagation via `pdepe` (SolveFP_*.m files)

The R translation is: `ks` for KDE, `qr.solve()` for the polynomial,
`pracma::fzero()` / `ks::kdde()` for critical points, `deSolve` + `ReacTran`
for Fokker-Planck. No MATLAB, no reticulate, no custom Rcpp.

The hard-coded things in the MATLAB that landscapeR must parameterise:
- Critical point locations (were hand-picked; must be automated via KDE peaks)
- Polynomial degree (hardcoded as 6; must be a parameter)
- KDE bandwidth (`outlierfactor` hand-tuned; must use Silverman or cross-validated)
- Diffusion coefficient D (constant; spatially varying is more realistic later)

## The HO-GSVD situation

Two R options found:
- `multiblock::hogsvd` — CRAN, active (v0.8.10, May 2026), standard formulation,
  requires full column rank
- Kempf HO-GSVD (SIAM 2022) — MATLAB only; no R package; handles rank-deficient
  matrices; this is the one the design spec cited explicitly

The diabetes genotype layer will be rank-deficient (p >> n). Whether
`multiblock::hogsvd` can handle it with pre-processing (truncated SVD projection)
is unknown — this is what Stage 0 thinness sweep with rank-deficiency axis decides.

`derekbeaton/GSVD` (GitHub only) is NOT the same thing — it's constrained/weighted
SVD (PCA/MCA generalization), not comparative GSVD. Different despite the name.
